## 🚀 ПОШАГОВЫЙ ЗАПУСК

### Шаг 1: Откройте Render Dashboard

Перейдите на: **https://dashboard.render.com**

Залогиньтесь через GitHub

### Шаг 2: Создайте новый Static Site

1. Нажмите кнопку **"New +"** (вверху справа)
2. Выберите **"Static Site"**
3. Подключите репозиторий:
   ```
   Repository: azizsalimov1998-del/Telegram-mini-app
   Branch: main
   ```

### Шаг 3: Настройте параметры

Заполните поля:

```
Name: safar-frontend
Region: Frankfurt (Germany)
Root Directory: s1-mini-app
Build Command: npm install && npm run build
Publish Directory: dist
```

### Шаг 4: Добавьте переменные окружения

В разделе **Environment** добавьте:

```env
Key: VITE_API_URL
Value: https://safar-backend.onrender.com/api
```

**Важно:** Замените `safar-backend.onrender.com` на ваш URL бэкенда!

### Шаг 5: Запустите деплой

1. Нажмите **"Create Static Site"**
2. Подождите 3-5 минут
3. Сборка завершится успешно

### Шаг 6: Скопируйте URL

После деплоя вы получите URL:
```
https://safar-frontend.onrender.com
```

## ⚙️ НАСТРОЙКА CORS НА БЭКЕНДЕ

После запуска фронтенда, нужно обновить CORS в бэкенде:

### В Render Dashboard (Backend):

1. Найдите ваш backend сервис
2. Перейдите в **Environment**
3. Добавьте/обновите переменную:

```env
Key: CORS_ORIGINS
Value: https://safar-frontend.onrender.com
```

4. Сохраните (автоматически пересоберется)

## 🐛 ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ

### Проблема 1: Сборка не удается

**Ошибка:** `npm install failed`

**Решение:**
```bash
# Проверьте package.json
cd s1-mini-app
npm install
npm run build

# Если локально работает, проблема в Render
# Проверьте логи в Render Dashboard
```

### Проблема 2: Пустая страница

**Причина:** Неверный VITE_API_URL

**Решение:**
1. Проверьте переменную окружения в Render
2. Убедитесь что бэкенд доступен
3. Проверьте консоль браузера

### Проблема 3: CORS ошибки

**Причина:** Бэкенд не разрешает фронтенд

**Решение:**
1. Обновите CORS_ORIGINS в бэкенде
2. Добавьте URL фронтенда
3. Перезапустите бэкенд

### Проблема 4: Товары не загружаются

**Причина:** Бэкенд недоступен или нет данных

**Решение:**
1. Проверьте что бэкенд работает
2. Проверьте базу данных
3. Добавьте тестовые данные
