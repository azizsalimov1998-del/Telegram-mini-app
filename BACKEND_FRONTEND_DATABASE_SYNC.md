## 🔧 КАК ЭТО РАБОТАЕТ

### Пример: Пользователь добавляет товар в корзину

1. **Frontend отправляет запрос:**
```javascript
POST https://safar-backend.onrender.com/api/cart/items
Body: { product_id: 1, quantity: 2 }
Headers: X-Telegram-Init-Data: ...
```

2. **Backend получает запрос:**
```python
@app.route('/api/cart/items', methods=['POST'])
def add_to_cart():
    data = request.json
    # Извлекаем user_id из Telegram initData
    user_id = extract_user_from_init_data(...)
    
    # Записываем в БАЗУ ДАННЫХ
    db.execute(
        'INSERT INTO cart_items (user_id, product_id, quantity) VALUES (?, ?, ?)',
        (user_id, data['product_id'], data['quantity'])
    )
    db.commit()
```

3. **База данных обновляется:**
```sql
INSERT INTO cart_items (user_id, product_id, quantity)
VALUES (12345, 1, 2);
```

4. **Другой пользователь видит изменения:**
```javascript
GET https://safar-backend.onrender.com/api/cart/items

// Backend читает из БД:
SELECT * FROM cart_items WHERE user_id = 12345;

// Возвращает:
{
  "items": [
    {"product_id": 1, "quantity": 2, ...}
  ]
}
```

## 💡 ВАЖНО ПОНИМАТЬ

### Frontend НЕ подключается к БД напрямую!

**НЕПРАВИЛЬНО:**
```
Frontend → SQLite file → Database  ❌
```

**ПРАВИЛЬНО:**
```
Frontend → API → Backend → Database  ✅
```

### Почему так:

1. **Безопасность:**
   - Клиенты не имеют прямого доступа к БД
   - Backend контролирует все операции
   - Валидация данных

2. **Архитектура:**
   - Static Sites не имеют доступа к файловой системе
   - Только HTTP запросы
   - Stateless архитектура

3. **Масштабируемость:**
   - Легко заменить БД на PostgreSQL
   - Можно добавить кэширование
   - Распределенная система

## 🐛 ВОЗМОЖНЫЕ ПРОБЛЕМЫ

### Проблема 1: Frontend не видит данные

**Симптомы:**
- Каталог пустой
- Ошибка загрузки

**Причина:**
- VITE_API_URL не установлен
- CORS ошибка
- Backend недоступен

**Решение:**
```env
# Frontend Environment:
VITE_API_URL = https://safar-backend.onrender.com/api

# Backend Environment:
CORS_ORIGINS = *
```

---

### Проблема 2: Корзина не сохраняется

**Симптомы:**
- Добавляете товар
- Обновляете страницу
- Корзина пустая

**Причина:**
- Backend не сохраняет в БД
- Проблема с авторизацией
- Нет user_id

**Решение:**
1. Проверьте логи Backend
2. Проверьте DISABLE_MINI_APP_AUTH
3. Проверьте Telegram initData

---

### Проблема 3: Разные базы данных

**Симптомы:**
- Локально работает
- На Render НЕ работает

**Причина:**
- Локальная БД ≠ БД на Render
- Разные пути к файлам

**Решение:**
- Используйте БД на Render через Backend
- Локально тоже отключите от Remote API
